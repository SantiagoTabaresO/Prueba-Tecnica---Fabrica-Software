**1. Coordinación del desarrollo:**  
Para coordinar el desarrollo con otra persona, estableceríamos los objetivos del proyecto con un plan inicial de funciones y diseño (back y front), definiríamos reuniones cada cierto tiempo para revisar avances y dudas, y usaríamos un repositorio compartido con ramas independientes para evitar conflictos. La comunicación se haría principalmente por comentarios en commits y un canal de chat del equipo.

**2. División de tareas:**  
- Persona A: Crear la estructura HTML y el diseño CSS de la página, incluyendo tarjetas, formulario, colores, aspecto y diseño visual.  
- Persona B: Implementar la lógica de JavaScript para crear, eliminar, filtrar y actualizar tareas, así como manejar la estructura de datos y eventos.

**3. Manejo de desacuerdos técnicos:**  
Para desacuerdos simples, discutiríamos las ventajas y desventajas de cada enfoque, buscaríamos consenso buscando algún punto en común, y en caso de persistir la diferencia, realizaríamos un prototipo rápido de ambas opciones para evaluar cuál se adapta mejor.

**4. Evidencia en commits claros:** 
- `funcionalidad: agregar funcionalidad de creación de tareas y validación de inputs`  
- `estilo: actualizar diseño de tarjetas y colores según estado de tarea`
